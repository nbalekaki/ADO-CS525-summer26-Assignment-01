1. Learning Objectives

By completing this assignment, students will be able to:

  • Implement disk-based page storage
  • Design and manage an in-memory buffer pool
  • Apply the Least Recently Used (LRU) replacement strategy
  • Safely manage dynamic memory in C
  • Understand why buffer management is critical for DBMS performance

2. Problem Overview

Modern database systems manage disk pages and memory buffers explicitly rather than relying on the operating system's paging mechanisms. In this assignment, you will implement:

  • A Storage Manager that manages fixed-size pages on disk
  • A Buffer Manager that caches disk pages in memory using an LRU replacement policy

Each page is 4096 bytes and is stored in a database file.

Extra credit is available for implementing alternative replacement strategies (e.g., CLOCK, LRU-K).

4. Specifications

4.1 Page Format

  • Page size: 4096 bytes
  • Pages are identified by a 0-based page number
  • Pages are always read and written in full

4.2 Storage Manager (SM)

The Storage Manager is responsible for reading and writing disk pages.

Required Functions

/* Initialization */
void initStorageManager(void);

/* File Operations */
RC createPageFile(char *fileName);
RC openPageFile(char *fileName, SM_FileHandle *fHandle);
RC closePageFile(SM_FileHandle *fHandle);
RC destroyPageFile(char *fileName);

/* Reading Blocks */
RC readBlock(int pageNum, SM_FileHandle *fHandle, SM_PageHandle memPage);
int getBlockPos(SM_FileHandle *fHandle);
RC readFirstBlock(SM_FileHandle *fHandle, SM_PageHandle memPage);
RC readPreviousBlock(SM_FileHandle *fHandle, SM_PageHandle memPage);
RC readCurrentBlock(SM_FileHandle *fHandle, SM_PageHandle memPage);
RC readNextBlock(SM_FileHandle *fHandle, SM_PageHandle memPage);
RC readLastBlock(SM_FileHandle *fHandle, SM_PageHandle memPage);

/* Writing Blocks */
RC writeBlock(int pageNum, SM_FileHandle *fHandle, SM_PageHandle memPage);
RC writeCurrentBlock(SM_FileHandle *fHandle, SM_PageHandle memPage);
RC appendEmptyBlock(SM_FileHandle *fHandle);
RC ensureCapacity(int numberOfPages, SM_FileHandle *fHandle);

Metadata to Maintain (SM_FileHandle)

  • File name
  • Total number of pages in the file
  • Current page position
  • Management info (file pointer or descriptor)

Requirements

  • Create the file if it does not exist (with one empty page)
  • Automatically extend the file when writing new pages
  • Use fseek, fread, and fwrite for file operations

4.3 Buffer Manager (BM)

The Buffer Manager caches disk pages in memory.

Buffer Pool

  • Fixed number of frames (configurable, e.g., 5)
  • Each frame holds exactly one page

Frame Metadata

  • Page number (which page is in this frame)
  • Dirty bit (has the page been modified?)
  • Pin count (fix count - how many clients are using this page)
  • LRU counter or equivalent tracking mechanism

Required Functions

/* Pool Handling */
RC initBufferPool(BM_BufferPool *const bm, const char *const pageFileName,
                  const int numPages, ReplacementStrategy strategy,
                  void *stratData);
RC shutdownBufferPool(BM_BufferPool *const bm);
RC forceFlushPool(BM_BufferPool *const bm);

/* Page Access */
RC pinPage(BM_BufferPool *const bm, BM_PageHandle *const page,
           const PageNumber pageNum);
RC unpinPage(BM_BufferPool *const bm, BM_PageHandle *const page);
RC markDirty(BM_BufferPool *const bm, BM_PageHandle *const page);
RC forcePage(BM_BufferPool *const bm, BM_PageHandle *const page);

/* Statistics */
PageNumber *getFrameContents(BM_BufferPool *const bm);
bool *getDirtyFlags(BM_BufferPool *const bm);
int *getFixCounts(BM_BufferPool *const bm);
int getNumReadIO(BM_BufferPool *const bm);
int getNumWriteIO(BM_BufferPool *const bm);

4.4 Replacement Policy

IMPORTANT:
  • Use Least Recently Used (LRU) replacement policy
  • Pinned pages must NOT be evicted (fixCount > 0)
  • Dirty pages MUST be written back to disk BEFORE eviction
  • Update LRU timestamp on EVERY pin operation (not just when loading from disk)

4.5 Statistics Functions

Implement functions that return statistics about:

  • getFrameContents: Returns array of page numbers in each frame (NO_PAGE for empty)
  • getDirtyFlags: Returns array of dirty flags for each frame
  • getFixCounts: Returns array of fix counts for each frame
  • getNumReadIO: Returns total number of pages read from disk
  • getNumWriteIO: Returns total number of pages written to disk

13. Common Mistakes to Avoid

  1. Not updating LRU on re-pin (costs 5-10 points)
    • Must update timestamp when pinning an already-buffered page
  2. Not writing dirty pages on eviction (costs 10-20 points)
    • Always check and write dirty pages before replacing
  3. Memory leaks (costs 10 points)
    • Free all allocated memory in shutdown
    • Use Valgrind: valgrind --leak-check=full ./test
  4. Evicting pinned pages (costs 5 points)
    • Only evict pages with fixCount == 0
  5. Not handling 100+ pages (costs 5-10 points)
    • Test with large files, small buffer pools
